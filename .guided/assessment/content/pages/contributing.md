## Page Purpose Statement

The Contributing page facilitates community engagement and continuous improvement, offering:
1. Five ways to contribute (from lightweight to heavy involvement)
2. Step-by-step process for submitting contributions
3. GitHub integration with links to repository, issues, and discussions
4. Open-source values and community guidelines

This page operationalizes the framework's "Open & Evolving" principle (Manifesto Principle 8) and builds an active community.

### 2. Five Ways to Contribute Section
- **Heading:** "How You Can Contribute"

**Method 1: Share Feedback**
- **Description:** Share your experience using the framework
- **How:** GitHub Discussions, Issues, or direct feedback
- **Examples:**
  - "Leveling criteria were helpful for our promotion cycle"
  - "W-Model topology fit our organization perfectly"
  - "Struggled with calibration process—here's what we learned"
- **Effort:** Low (15-30 minutes)
- **Link:** GitHub Discussions

**Method 2: Report Issues or Suggest Improvements**
- **Description:** Identify problems, gaps, or enhancement opportunities
- **How:** Open a GitHub Issue
- **Examples:**
  - "Progression Pillar examples are too software-focused"
  - "Missing guidance for non-engineering roles"
  - "Implementation timeline is unrealistic for small companies"
- **Effort:** Low (30 minutes)
- **Link:** GitHub Issues

**Method 3: Add Examples or Case Studies**
- **Description:** Share how your organization adapted the framework
- **How:** Submit pull request with case study or example
- **Examples:**
  - "How we implemented W-Model in a 50-person startup"
  - "Adapting proficiency pillars for data science roles"
  - "Lessons learned from our first calibration cycle"
- **Effort:** Medium (2-4 hours)
- **Link:** GitHub repository

**Method 4: Improve Documentation**
- **Description:** Fix typos, clarify content, add missing sections
- **How:** Fork repository, edit markdown files, submit pull request
- **Examples:**
  - Fix typos or broken links
  - Improve clarity of confusing sections
  - Add FAQ entries
  - Translate content to other languages
- **Effort:** Medium (1-3 hours)
- **Link:** GitHub repository, Contributing guidelines

**Method 5: Extend the Framework**
- **Description:** Propose new framework components or major enhancements
- **How:** Open GitHub Discussion first (align before building), then submit pull request
- **Examples:**
  - New topology model (e.g., "Matrix Model")
  - Additional progression pillars for specialized roles
  - Tools for assessment or tracking
  - Research papers or data analysis
- **Effort:** High (days to weeks)
- **Link:** GitHub Discussions (for alignment), then repository

### 3. Contribution Process Section
- **Heading:** "How to Submit a Contribution"
- **4-Step Process:**

**Step 1: Review Existing Issues and Discussions**
- Check if your idea or issue already exists
- Join existing conversations if relevant
- Links: GitHub Issues, GitHub Discussions

**Step 2: Choose Contribution Method**
- Lightweight feedback → GitHub Discussion
- Bug/issue → GitHub Issue
- Documentation/examples → Fork and PR
- Major extension → Discussion first, then PR

**Step 3: Follow Contribution Guidelines**
- Respect Code of Conduct (link to CoC if available)
- Follow documentation style guide
- Provide clear descriptions and context
- Test changes before submitting (if applicable)

**Step 4: Submit and Engage**
- Submit GitHub Issue, Discussion, or Pull Request
- Respond to questions and feedback from maintainers
- Iterate based on review
- Celebrate when merged!

## User Journeys Touching This Page

### 1. Wanting to Give Feedback (User with Experience)
- **Entry:** Used framework, wants to share experience
- **Actions on Contributing:**
  - Read "Share Feedback" contribution method
  - Click through to GitHub Discussions
  - Post feedback or testimonial
- **Exit:** GitHub Discussions (post feedback), feel connected to community

### 2. Identifying an Issue (Critical User)
- **Entry:** Found a problem or gap in framework
- **Actions on Contributing:**
  - Read "Report Issues or Suggest Improvements"
  - Click through to GitHub Issues
  - Open new issue with description
- **Exit:** GitHub Issues (submit issue), await response from maintainers

### 3. Wanting to Contribute Meaningfully (Engaged User)
- **Entry:** Loves framework, wants to help improve it
- **Actions on Contributing:**
  - Review all five contribution methods
  - Assess effort level and fit
  - Check GitHub Issues for "Good First Issues"
  - Fork repository and submit pull request
- **Exit:** GitHub repository (contribute), become active community member

### 4. Evaluating Community Health (Potential Adopter)
- **Entry:** Considering framework adoption, assessing community activity
- **Actions on Contributing:**
  - Check GitHub links to see recent activity
  - Browse open issues and discussions
  - Assess responsiveness of maintainers
  - Evaluate contribution volume and diversity
- **Exit:** Increased confidence in framework (active community) or concerns (inactive community)

## Notes

- **GitHub-Centric:** All contributions happen on GitHub (not email, forms, etc.)
- **Low Barrier to Entry:** Method 1 (Share Feedback) requires no technical skills
- **Multiple Contribution Types:** Not just code—feedback, examples, docs all valued
- **Community Health Indicator:** Active contributions signal healthy, maintained project
- **Alignment Before Big Work:** "Discussion first" for major extensions prevents wasted effort
- **Recognition Matters:** Acknowledging contributors builds community and motivation
- **Code of Conduct Essential:** Respectful community guidelines prevent toxic behavior
- **Good First Issues:** Tagging beginner-friendly issues lowers contribution barrier
- **Virtuous Cycle:** Users → Adopters → Contributors → Maintainers (pathway to deep engagement)
- **Version Control Benefit:** GitHub enables transparent change tracking (supports "Open & Evolving")
- **Fork and Adapt:** MIT license allows organizations to fork for private use (not just contribute back)
- **Community Over Perfection:** Encouraging contributions even if not perfect (iterate together)
